# mySTL 调试宏系统设计指南

## 概述

本文档旨在为 mySTL 项目提供统一的调试宏定义和使用指南，解决当前项目中关于 `NDEBUG` 宏使用的疑问，并建立一套完整的调试和断言系统。

## 1. NDEBUG 宏的C/C++标准定义

### 1.1 标准规定
根据 C/C++ 标准，`NDEBUG` 是一个特殊的宏：
- **C标准**：在 `<assert.h>` 中定义，当定义了 `NDEBUG` 时，`assert` 宏不执行任何操作
- **C++标准**：在 `<cassert>` 中定义，行为与C标准相同
- **默认状态**：默认情况下 `NDEBUG` 未定义，`assert` 会执行运行时检查

### 1.2 编译器行为
不同编译器对 `NDEBUG` 的处理：
- **GCC/Clang**：通常通过 `-DNDEBUG` 编译选项定义
- **MSVC**：通过 `/DNDEBUG` 编译选项定义
- **CMake**：在 Release 构建中自动定义 `NDEBUG`

## 3. 设计建议

### 3.1 方案一：使用标准 NDEBUG（推荐）
**优点**：
- 符合C/C++标准
- 与现有生态系统兼容
- 编译器自动支持

**实现方式**：
```cpp
// 在需要调试检查的地方
#ifndef NDEBUG
// 调试代码
#endif
```

**构建配置**：
```bash
# Debug 构建（默认）
g++ -std=c++17 -I./include -o test test.cpp

# Release 构建
g++ -std=c++17 -DNDEBUG -I./include -o test test.cpp
```

### 3.2 方案二：自定义调试宏
**优点**：
- 更细粒度的控制
- 可以定义多个调试级别

**实现方式**：
```cpp
// config.hpp
#pragma once

#ifndef MYSTL_DEBUG
#ifdef NDEBUG
#define MYSTL_DEBUG 0
#else
#define MYSTL_DEBUG 1
#endif
#endif

// 调试级别
#ifndef MYSTL_DEBUG_LEVEL
#define MYSTL_DEBUG_LEVEL 1
#endif
```

## 4. 完整的调试宏系统设计

### 4.1 配置头文件
创建 `include/config.hpp`：

```cpp
#pragma once

/**
 * @file config.hpp
 * @brief mySTL 项目配置和调试宏定义
 */

// ============================================================================
// 调试配置
// ============================================================================

/**
 * @brief 主调试宏
 * 
 * 当定义了 NDEBUG 时，MYSTL_DEBUG 为 0
 * 否则 MYSTL_DEBUG 为 1
 */
#ifndef MYSTL_DEBUG
#ifdef NDEBUG
#define MYSTL_DEBUG 0
#else
#define MYSTL_DEBUG 1
#endif
#endif

/**
 * @brief 调试级别
 * 
 * 0: 无调试
 * 1: 基本调试（断言、基本检查）
 * 2: 详细调试（额外检查、日志）
 * 3: 完整调试（所有检查、详细日志）
 */
#ifndef MYSTL_DEBUG_LEVEL
#define MYSTL_DEBUG_LEVEL 1
#endif

/**
 * @brief 断言启用
 */
#ifndef MYSTL_ENABLE_ASSERT
#define MYSTL_ENABLE_ASSERT MYSTL_DEBUG
#endif

/**
 * @brief 边界检查启用
 */
#ifndef MYSTL_ENABLE_BOUNDS_CHECK
#define MYSTL_ENABLE_BOUNDS_CHECK MYSTL_DEBUG
#endif

/**
 * @brief 类型检查启用
 */
#ifndef MYSTL_ENABLE_TYPE_CHECK
#define MYSTL_ENABLE_TYPE_CHECK MYSTL_DEBUG
#endif

// ============================================================================
// 断言宏
// ============================================================================

/**
 * @brief 基本断言宏
 */
#if MYSTL_ENABLE_ASSERT
#include <cassert>
#define MYSTL_ASSERT(expr) assert(expr)
#define MYSTL_ASSERT_MSG(expr, msg) \
    do { \
        if (!(expr)) { \
            std::cerr << "Assertion failed: " << #expr << "\n" << msg << "\n"; \
            std::cerr << "File: " << __FILE__ << ", Line: " << __LINE__ << "\n"; \
            std::abort(); \
        } \
    } while (false)
#else
#define MYSTL_ASSERT(expr) ((void)0)
#define MYSTL_ASSERT_MSG(expr, msg) ((void)0)
#endif

/**
 * @brief 调试专用断言（仅在详细调试级别启用）
 */
#if MYSTL_DEBUG_LEVEL >= 2
#define MYSTL_DEBUG_ASSERT(expr) MYSTL_ASSERT(expr)
#else
#define MYSTL_DEBUG_ASSERT(expr) ((void)0)
#endif

// ============================================================================
// 调试检查宏
// ============================================================================

/**
 * @brief 边界检查
 */
#if MYSTL_ENABLE_BOUNDS_CHECK
#define MYSTL_BOUNDS_CHECK(ptr, size, index) \
    MYSTL_ASSERT_MSG((index) < (size), \
        "Index " << (index) << " out of bounds (size=" << (size) << ")")
#else
#define MYSTL_BOUNDS_CHECK(ptr, size, index) ((void)0)
#endif

/**
 * @brief 空指针检查
 */
#if MYSTL_DEBUG
#define MYSTL_NULL_CHECK(ptr) \
    MYSTL_ASSERT_MSG((ptr) != nullptr, "Null pointer detected")
#else
#define MYSTL_NULL_CHECK(ptr) ((void)0)
#endif

/**
 * @brief 对齐检查
 */
#if MYSTL_DEBUG
#define MYSTL_ALIGN_CHECK(ptr, alignment) \
    MYSTL_ASSERT_MSG((reinterpret_cast<uintptr_t>(ptr) & ((alignment) - 1)) == 0, \
        "Pointer " << (ptr) << " not aligned to " << (alignment))
#else
#define MYSTL_ALIGN_CHECK(ptr, alignment) ((void)0)
#endif

// ============================================================================
// 调试日志宏
// ============================================================================

/**
 * @brief 调试日志
 */
#if MYSTL_DEBUG_LEVEL >= 2
#include <iostream>
#define MYSTL_DEBUG_LOG(msg) \
    std::cerr << "[DEBUG] " << __FILE__ << ":" << __LINE__ << " - " << msg << "\n"
#else
#define MYSTL_DEBUG_LOG(msg) ((void)0)
#endif

#if MYSTL_DEBUG_LEVEL >= 3
#define MYSTL_TRACE_LOG(msg) \
    std::cerr << "[TRACE] " << __FILE__ << ":" << __LINE__ << " - " << msg << "\n"
#else
#define MYSTL_TRACE_LOG(msg) ((void)0)
#endif

// ============================================================================
// 编译时检查
// ============================================================================

/**
 * @brief 静态断言（C++11及以上）
 */
#define MYSTL_STATIC_ASSERT(expr, msg) static_assert((expr), msg)

/**
 * @brief 类型特征检查
 */
#if MYSTL_ENABLE_TYPE_CHECK
#define MYSTL_TYPE_CHECK(T, trait) \
    static_assert(trait<T>::value, #T " does not satisfy " #trait)
#else
#define MYSTL_TYPE_CHECK(T, trait) ((void)0)
#endif
```

```cpp

T& vector<T>::at(size_type pos) {
    MYSTL_BOUNDS_CHECK(data_, size_, pos);
    return data_[pos];
}

// 在算法中的调试日志
template<typename InputIt, typename T>
InputIt find(InputIt first, InputIt last, const T& value) {
    MYSTL_DEBUG_LOG("find called with range size: " << std::distance(first, last));
    for (; first != last; ++first) {
        if (*first == value) {
            MYSTL_DEBUG_LOG("find found value at position: " << std::distance(first, last));
            return first;
        }
    }
    MYSTL_DEBUG_LOG("find did not find value");
    return last;
}
```

## 7. 最佳实践

### 7.1 使用指南
1. **生产代码**：使用 `MYSTL_ASSERT` 进行关键检查
2. **调试代码**：使用 `MYSTL_DEBUG_ASSERT` 进行额外检查
3. **边界检查**：在访问容器元素时使用 `MYSTL_BOUNDS_CHECK`
4. **指针操作**：在解引用前使用 `MYSTL_NULL_CHECK`
5. **内存操作**：在分配/释放内存时使用 `MYSTL_ALIGN_CHECK`

### 7.2 注意事项
1. **副作用**：断言表达式不应有副作用
   ```cpp
   // 错误：有副作用
   MYSTL_ASSERT(++x > 0);
   
   // 正确：无副作用
   ++x;
   MYSTL_ASSERT(x > 0);
   ```

2. **性能影响**：在性能关键路径中谨慎使用调试检查

3. **错误信息**：使用 `MYSTL_ASSERT_MSG` 提供有意义的错误信息

### 7.3 代码审查要点
1. 检查所有调试宏的使用是否正确
2. 确保没有在 Release 构建中引入性能问题
3. 验证错误信息清晰有用
