# 迭代器适配器系统讲解

## 概述

迭代器适配器是C++标准库中强大的设计模式实现，它们通过包装现有的迭代器或容器接口，提供新的迭代行为或功能。mySTL项目实现了完整的迭代器适配器系统，遵循C++标准库的设计理念，同时提供了详细的学习和实现参考。

## 迭代器适配器分类

### 1. 方向适配器
**改变迭代方向**
- **`reverse_iterator`**：反向迭代器，将正向遍历转换为反向遍历
- **应用场景**：需要从后向前遍历容器，查找最后一个匹配元素，检查回文等

### 2. 插入适配器
**将赋值操作转换为插入操作**
- **`back_insert_iterator`**：尾部插入迭代器，总是插入到容器末尾
- **`front_insert_iterator`**：头部插入迭代器，总是插入到容器开头  
- **`insert_iterator`**：通用插入迭代器，在指定位置插入
- **应用场景**：算法输出到容器，数据重组，批量插入等

### 3. 流适配器
**连接迭代器与I/O流**
- **`istream_iterator`**：输入流迭代器，从输入流读取数据
- **`ostream_iterator`**：输出流迭代器，向输出流写入数据
- **应用场景**：文件处理，控制台I/O，数据序列化等

### 4. 移动适配器
**支持移动语义**
- **`move_iterator`**：移动迭代器，将解引用转换为移动操作
- **应用场景**：高效数据转移，避免不必要的拷贝

## 设计模式分析

### 适配器模式（Adapter Pattern）
迭代器适配器是适配器模式的经典应用：

```cpp
// 适配器模式结构
template<typename Iterator>
class IteratorAdapter {
private:
    Iterator base_iterator_;  // 被适配的迭代器
    
public:
    // 提供新的接口，内部调用base_iterator_的方法
    value_type operator*() const {
        // 转换操作
        return transform(*base_iterator_);
    }
    
    // 其他操作类似...
};
```

### 代理模式（Proxy Pattern）
许多迭代器适配器也使用了代理模式：

```cpp
// 代理模式：控制对底层对象的访问
class IteratorProxy {
private:
    UnderlyingIterator& real_iterator_;
    
public:
    // 代理操作，可以添加额外逻辑
    reference operator*() const {
        // 可以添加缓存、验证、转换等逻辑
        return *real_iterator_;
    }
};
```

### 空对象模式（Null Object Pattern）
输出迭代器（如插入迭代器）使用空对象模式：

```cpp
// 空操作：operator* 和 operator++ 返回自身
insert_iterator& operator*() { return *this; }
insert_iterator& operator++() { return *this; }
insert_iterator& operator++(int) { return *this; }
```

## 核心实现原理

### 1. 类型萃取（Type Traits）
所有迭代器适配器都使用 `iterator_traits` 来获取底层迭代器的类型信息：

```cpp
template<typename Iterator>
class reverse_iterator {
public:
    using iterator_type = Iterator;
    using iterator_category = typename iterator_traits<Iterator>::iterator_category;
    using value_type = typename iterator_traits<Iterator>::value_type;
    using difference_type = typename iterator_traits<Iterator>::difference_type;
    using pointer = typename iterator_traits<Iterator>::pointer;
    using reference = typename iterator_traits<Iterator>::reference;
};
```

### 2. 迭代器类别保持
适配器保持或转换底层迭代器的类别：

| 适配器 | 输入类别 | 输出类别 | 说明 |
|--------|----------|----------|------|
| `reverse_iterator` | 保持原类别 | 同输入 | 方向反转，类别不变 |
| 插入迭代器 | 任意 | `output_iterator_tag` | 转换为输出迭代器 |
| 流迭代器 | `input_iterator_tag` | `output_iterator_tag` | 特定I/O操作 |
| `move_iterator` | 保持原类别 | 同输入 | 添加移动语义 |

### 3. 操作符重载策略
不同适配器采用不同的操作符重载策略：

```cpp
// 反向迭代器：反转方向
reference operator*() const {
    Iterator tmp = current_;
    return *--tmp;  // 返回前一个元素
}

// 插入迭代器：转换赋值为插入
insert_iterator& operator=(const value_type& value) {
    iter_ = container_->insert(iter_, value);
    ++iter_;  // 移动到新插入元素之后
    return *this;
}

// 移动迭代器：转换为移动操作
value_type&& operator*() const {
    return std::move(*current_);
}
```

void safe_iteration(Iterator begin, Iterator end) {
    if (begin == end) return;  // 空范围
    
    try {
        for (auto it = begin; it != end; ++it) {
            // 安全操作
            process(*it);
        }
    } catch (const std::exception& e) {
        // 异常处理
        handle_error(e);
    }

        throw std::out_of_range("Invalid iterator position");
    }
    return mystl::inserter(c, pos);
}

## 具体适配器详解

### 1. reverse_iterator（反向迭代器）
**核心特性**：
- 方向反转：`++` 对应底层迭代器的 `--`
- 解引用偏移：`operator*` 返回 `current-1`
- 保持所有原始迭代器特性

**关键实现**：
```cpp
reference operator*() const {
    Iterator tmp = current_;
    return *--tmp;
}

reverse_iterator& operator++() {
    --current_;
    return *this;
}
```

### 2. 插入迭代器家族
**共同特点**：
- 输出迭代器类别（`output_iterator_tag`）
- 空操作：`operator*` 和 `operator++` 返回自身
- 核心操作：`operator=` 执行插入

**区别**：
- `back_insert_iterator`：`push_back`
- `front_insert_iterator`：`push_front`  
- `insert_iterator`：`insert` + 位置管理

### 3. 流迭代器
**输入流迭代器**：
- 从流读取直到EOF
- 默认构造表示流结束
- 单趟迭代（输入迭代器）

**输出流迭代器**：
- 向流写入数据
- 可指定分隔符
- 输出迭代器类别

### 4. move_iterator
**C++11新特性**：
- 将拷贝转换为移动
- 提高大型对象或资源句柄的转移效率
- 保持原始迭代器所有操作

## 常见问题与解决方案

### Q1: 迭代器适配器会导致性能损失吗？
**A**: 通常不会。迭代器适配器是零开销抽象：
- 所有操作都是内联的
- 无额外内存分配
- 编译时类型检查
- 运行时无额外开销

### Q2: 如何选择正确的插入迭代器？
**A**: 根据插入位置需求：
- **末尾插入**：`back_insert_iterator`（最常用）
- **开头插入**：`front_insert_iterator`（需要容器支持`push_front`）
- **任意位置插入**：`insert_iterator`（最灵活）

### Q3: 反向迭代器的`base()`方法返回什么？
**A**: `base()`返回底层正向迭代器，但需要注意：
```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
auto rit = vec.rbegin();  // 指向5
auto it = rit.base();     // 指向end()（5之后）

// 关系：&*(rit) == &*(it - 1)
```

### Q4: 流迭代器何时失效？
**A**: 
- `istream_iterator`：流到达EOF或发生错误时失效
- `ostream_iterator`：底层流关闭或发生错误时失效
- 默认构造的流迭代器表示流结束

### Q5: 移动迭代器使用后源数据会怎样？
**A**: 源数据处于有效但未指定状态：
- 可以安全析构
- 可以重新赋值
- 不应再读取其值（除非重新赋值）

## 扩展与自定义

### 1. 自定义迭代器适配器
```cpp
// 示例：过滤迭代器
template<typename Iterator, typename Predicate>
class filter_iterator {
private:
    Iterator current_;
    Iterator end_;
    Predicate pred_;
    
    void advance_to_valid() {
        while (current_ != end_ && !pred_(*current_)) {
            ++current_;
        }
    }
    
public:
    filter_iterator(Iterator begin, Iterator end, Predicate pred)
        : current_(begin), end_(end), pred_(pred) {
        advance_to_valid();
    }
    
    auto operator*() const { return *current_; }
    
    filter_iterator& operator++() {
        ++current_;
        advance_to_valid();
        return *this;
    }
    
    // 其他必要操作...
};

// 使用示例
auto is_even = [](int x) { return x % 2 == 0; };
auto fit = filter_iterator(vec.begin(), vec.end(), is_even);
```
