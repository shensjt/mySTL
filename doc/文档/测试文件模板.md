# 测试文件模板 / Test File Template

## 概述 / Overview

本文档提供了一个通用的测试文件模板，适用于 mySTL 项目中所有组件的测试文件。该模板遵循最佳实践，包括完整的 Doxygen 注释、清晰的测试组织结构和全面的错误处理。

This document provides a general test file template suitable for all component test files in the mySTL project. The template follows best practices, including complete Doxygen comments, clear test organization structure, and comprehensive error handling.

## 模板结构 / Template Structure

### 1. 文件头注释 / File Header Comments

```cpp
/**
 * @file test_component_name.cpp
 * @brief Comprehensive test suite for mystl::component_name implementation
 * @brief mystl::component_name 实现的全面测试套件
 * 
 * This file contains extensive tests for the [component name] library in mystl.
 * It verifies the correctness of [component functionality] against expected behavior.
 * 
 * 此文件包含对 mystl 中 [组件名称] 库的广泛测试。
 * 它根据预期行为验证 [组件功能] 的正确性。
 * 
 * The tests are organized into logical sections corresponding to different
 * categories of functionality, making it easy to locate and understand specific tests.
 * 
 * 测试按功能的不同类别组织成逻辑部分，便于定位和理解特定测试。
 * 
 * @author shensjt
 * @date [YYYY-MM-DD]
 * 
 * @see include/[path]/component_name.hpp
 * @see 参考：include/[path]/component_name.hpp
 */
```

### 2. 包含头文件和宏定义 / Include Headers and Macro Definitions

```cpp
#include "../include/[path]/component_name.hpp"
#include <iostream>
#include <cassert>

// Helper macro for testing / 测试辅助宏
#define TEST(expr) \
    do { \
        if (!(expr)) { \
            std::cerr << "Test failed at line " << __LINE__ << ": " << #expr << std::endl; \
            std::cerr << "测试失败，行号 " << __LINE__ << ": " << #expr << std::endl; \
            assert(false && #expr); \
        } \
    } while(0)

// Alternative: More detailed test macro with test name / 替代方案：带测试名称的详细测试宏
#define TEST_NAMED(name, expr) \
    do { \
        if (!(expr)) { \
            std::cerr << "Test '" << name << "' failed at line " << __LINE__ << ": " << #expr << std::endl; \
            std::cerr << "测试 '" << name << "' 失败，行号 " << __LINE__ << ": " << #expr << std::endl; \
            assert(false && #expr); \
        } \
    } while(0)
```

## 测试组织最佳实践 / Test Organization Best Practices

### 1. 逻辑分组 / Logical Grouping
- 将相关测试组织在逻辑部分中 / Organize related tests in logical sections
- 使用有意义的章节标题 / Use meaningful section titles
- 每个部分测试一个特定的功能区域 / Each section tests a specific functional area

### 2. 测试命名 / Test Naming
- 使用描述性的测试名称 / Use descriptive test names
- 在注释中解释测试的目的 / Explain the purpose of the test in comments
- 对于复杂的测试，添加设置、执行、验证的注释 / For complex tests, add setup, exercise, verify comments

### 3. 错误处理 / Error Handling
- 使用 `TEST` 宏进行断言 / Use `TEST` macro for assertions
- 对于异常测试，使用 try-catch 块 / For exception testing, use try-catch blocks
- 提供有意义的错误消息 / Provide meaningful error messages

### 4. 资源管理 / Resource Management
- 在作用域块中管理测试资源 / Manage test resources in scope blocks
- 使用 RAII 原则 / Use RAII principles
- 避免资源泄漏 / Avoid resource leaks

### 5. 可读性 / Readability
- 保持测试代码简洁 / Keep test code concise
- 使用辅助函数减少重复 / Use helper functions to reduce duplication
- 添加注释解释复杂的测试逻辑 / Add comments to explain complex test logic

## 示例：完整的测试文件 / Example: Complete Test File

以下是一个基于模板的完整示例 / Here is a complete example based on the template:

```cpp
/**
 * @file test_vector.cpp
 * @brief Comprehensive test suite for mystl::vector implementation
 * @brief mystl::vector 实现的全面测试套件
 * 
 * This file contains extensive tests for the vector container in mystl.
 * It verifies the correctness of vector operations including construction,
 * element access, modification, and iterator functionality.
 * 
 * 此文件包含对 mystl 中 vector 容器的广泛测试。
 * 它验证 vector 操作的正确性，包括构造、元素访问、修改和迭代器功能。
 * 
 * The tests are organized into logical sections corresponding to different
 * aspects of vector functionality.
 * 
 * 测试按 vector 功能的不同方面组织成逻辑部分。
 * 
 * @author mySTL Team / mySTL 团队
 * @date 2026-03-21 / 2026年3月21日
 * 
 * @see include/mystl/vector.hpp
 * @see 参考：include/mystl/vector.hpp
 */

#include "../include/mystl/vector.hpp"
#include <iostream>
#include <cassert>

#define TEST(expr) \
    do { \
        if (!(expr)) { \
            std::cerr << "Test failed at line " << __LINE__ << ": " << #expr << std::endl; \
            std::cerr << "测试失败，行号 " << __LINE__ << ": " << #expr << std::endl; \
            assert(false && #expr); \
        } \
    } while(0)

/**
 * @brief Test vector constructors
 * @brief 测试 vector 构造函数
 */
void test_vector_constructors() {
    std::cout << "Testing vector constructors... / 测试 vector 构造函数..." << std::endl;
    
    // 默认构造函数 / Default constructor
    mystl::vector<int> v1;
    TEST(v1.empty());
    TEST(v1.size() == 0);
    
    // 指定大小的构造函数 / Constructor with specified size
    mystl::vector<int> v2(5, 42);
    TEST(v2.size() == 5);
    for (size_t i = 0; i < v2.size(); ++i) {
        TEST(v2[i] == 42);
    }
    
    std::cout << "  All constructor tests passed! / 所有构造函数测试通过！" << std::endl;
}

/**
 * @brief Main test function
 * @brief 主测试函数
 */
int main() {
    std::cout << "=== Testing mystl vector === / === 测试 mystl vector ===" << std::endl;
    std::cout << std::endl;
    
    try {
        test_vector_constructors();
        // ... 更多测试函数调用 / More test function calls
    } catch (const std::exception& e) {
        std::cerr << "Exception caught during testing: " << e.what() << std::endl;
        std::cerr << "测试过程中捕获异常: " << e.what() << std::endl;
        return 1;
    } catch (...) {
        std::cerr << "Unknown exception caught during testing" << std::endl;
        std::cerr << "测试过程中捕获未知异常" << std::endl;
        return 1;
    }
    
    std::cout << "\n=== All tests passed! / 所有测试通过！ ===" << std::endl;
    return 0;
}
```

## 注意事项 / Notes

1. **测试独立性** / **Test Independence**: 每个测试应该独立于其他测试 / Each test should be independent of other tests
2. **可重复性** / **Repeatability**: 测试应该产生相同的结果每次运行 / Tests should produce the same results each time they run
3. **全面性** / **Comprehensiveness**: 覆盖正常情况、边界情况和错误情况 / Cover normal cases, edge cases, and error cases
4. **性能** / **Performance**: 避免在测试中引入不必要的性能开销 / Avoid introducing unnecessary performance overhead in tests
5. **可维护性** / **Maintainability**: 保持测试代码的清晰和可维护 / Keep test code clear and maintainable
