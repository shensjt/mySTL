## 项目概述

### 项目名称
mySTL - 教育性C++标准模板库实现项目

### 核心目标
通过亲手实现STL核心组件，深入理解C++模板元编程、容器设计、算法优化等高级主题。

### 项目特点
1. **教育优先**：理解原理比完成功能更重要
2. **循序渐进**：从简单到复杂，逐步挑战
3. **测试驱动**：先写测试，再实现功能
4. **文档完整**：记录学习过程和设计决策
5. **标准对齐**：尽量对齐标准库实现，除非难以实现

## 实现原则

### 1. 标准对齐原则
- **接口兼容**：尽量与C++标准库接口保持一致
- **行为一致**：在相同输入下产生与标准库相同的行为
- **异常安全**：提供与标准库相同的异常安全保证
- **性能要求**：时间复杂度与标准库一致，实际性能尽量接近

### 2. 实现优先级
1. **正确性**：功能正确，无bug
2. **可读性**：代码清晰，易于理解
3. **性能**：在保证正确性的前提下优化性能
4. **兼容性**：支持不同编译器和平台

### 3. 代码质量要求
- **编码规范**：
  - 类型命名：PascalCase（如：MyVector, IteratorTraits）
  - 函数命名：snake_case（如：push_back, find_first_of）
  - 变量命名：snake_case（如：element_count, data_pointer）
  - 模板参数：PascalCase（如：ValueType, Allocator）
  - 私有成员：后缀下划线（如：data_, size_）

- **注释要求**：
  - 所有头文件必须有中英双语注释
  - 所有公有接口必须有详细文档注释
  - 复杂算法必须有算法原理注释
  - 关键设计决策必须有设计注释

### 4. 测试要求
- **测试类型**：
  - 功能测试：验证基本功能正确性
  - 边界测试：测试空容器、最大容量等边界情况
  - 异常测试：验证异常安全保证
  - 性能测试：验证时间复杂度保证
  - 兼容性测试：与标准库行为对比

- **测试文件命名**：
  - `test_<组件名>.cpp`（如：test_vector.cpp）
  - `benchmark_<组件名>.cpp`（性能测试）
